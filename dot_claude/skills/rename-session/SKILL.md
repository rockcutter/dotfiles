# rename-session

カレントセッションの表示名を変更する。agent view やセッション一覧で、内容がひと目で分かる名前を付けるのが目的。

## 手順

1. 命名規則に従って新しいセッション名を決める
2. このスキルの base directory 配下のスクリプトを実行する:

   ```bash
   <base-directory>/scripts/rename_session.sh "<新しい名前>"
   ```

3. スクリプトの出力（`updated:` の行）で書き換えられたファイルを確認し、新しい名前をユーザーに報告する。失敗した場合はエラー内容をそのまま報告する

## 命名規則

形式: `<タスク名>_<現在の状況>`（日本語、アンダースコア区切り）

- タスク名: このセッションの主タスクを表す短い名詞句
- 現在の状況: いまのフェーズを表す短い語。例: 調査中 / 実装中 / レビュー待ち / ブロック中 / 完了
- 全体で30文字以内が目安。改行・スラッシュ・引用符は使わない

引数の扱い:

- 引数なし → 会話の内容からタスク名と現在の状況を判断して命名する
- 引数が「タスク名_状況」の完全な形 → そのまま使う
- 引数がタスク名だけ → 現在の状況を判断して `_<状況>` を付け足す

例:

- `nvim設定追加_実装中`
- `デプロイ調査_ブロック中`
- `rename検証_完了`

## 仕組みと注意

スクリプトは環境変数 `CLAUDE_CODE_SESSION_ID` で自セッションを特定し、以下の `name` を書き換える。`nameSource` を `user` にすることで、自動命名による上書きを防ぐ。

- `$CLAUDE_JOB_DIR/state.json` — bgジョブの場合のみ存在。agent view のbgジョブ行の表示元で、ここを書き換えるとジョブ再起動後も名前が引き継がれる
- `~/.claude/sessions/<pid>.json` — 実行中セッションのレジストリ。`claude agents --json` および agent view の表示元

注意点:

- どちらも Claude Code の非公開の内部ファイルであり、バージョンアップで構造が変わると動かなくなる可能性がある。スクリプトが「no session file found」等で失敗した場合は、`~/.claude/sessions/` と `$CLAUDE_JOB_DIR/state.json` の構造が変わっていないか確認する
- interactive セッションでは、セッション自身のプロンプトバー表示への反映や、セッション終了後の名前の永続性は保証されない。ライブのセッション一覧表示を整える用途と考える
- subagent 内から実行すると subagent 自身のセッションが対象になり、意図した rename にならない。メインの会話から実行すること
